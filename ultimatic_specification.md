## 1. Introduction

The Ultimatic keyer mode is a historic iambic keyer implementation from 1955 that uses a **squeeze technique** to minimize the number of paddle contacts needed to send CW characters. The operator holds the first paddle continuously and "squeezes" (briefly presses) the second paddle to alternate between dits and dahs.

**Key advantages:**
- Minimized paddle movements
- Natural squeeze technique
- Memory and seizure mechanisms guarantee proper timing

**Implementation speed for examples:** 15 WPM
- 1 dit unit = 80ms
- 1 dah = 3 units = 240ms  
- Element spacing = 1 unit = 80ms

## 2. Original Specification from QST 1955

### Summary of Actions of the Keys

1. A single character is generated by momentary or prolonged closure of a key. The character is held by the memory for a positive time-base pulse if the key is released prior to that pulse.

2. Successive like characters are generated by constant closure of a key.

3. When one memory is already actuated, closure of the opposite key before generation of the first-stored character activates the opposite memory. The firstly actuated memory retains control of the sequencer until one character of its type is delivered at the output. The secondly actuated memory then assumes control of the sequencer (as the first key is open or still closed) and the next output character is of the second type.

4. Continued closure of this second key maintains control of the sequencer after the transfer action, and the output is a series of the second type until that key is released. This obtains even with the first key still closed.

5. Release of the second key, with the first key still closed, causes the output to revert to the first character type.

6. Release and reclosure of the first key (just a flick!) reactuates the first memory and seizes control of the sequencer — the second key closed all the while — and the output reverts to the first character type until that first key is again released or until the opposite type character is flicked in by the second key. At least one character of the first type is guaranteed by the memory.

7. In recapitulation, any closure of a key guarantees at least one character of that type, transmitted in correct relationship to the order of closure, regardless of intervening selective motions. Whenever a key makes contact, the output subsequent to the character in progress corresponds to that key until the other key makes contact or the first key is released.

## 3. Fundamental Rules

### Rule 1: Memory Flag (Point 1)
- Any paddle press (rising edge) sets the corresponding memory flag
- Memory guarantees at least ONE element will be sent
- Memory is cleared after element transmission begins

### Rule 2: Continuous Operation (Point 2)
- Holding a paddle after its element completes → continues sending that element type

### Rule 3: Seizure (Point 3)
- When first memory active, pressing second paddle activates second memory
- First element MUST complete before control transfers
- Second memory then assumes control (regardless of first paddle state)

### Rule 4: Maintained Control (Point 4)
- Holding second paddle after seizure → series of second element type
- This works even with first paddle still held

### Rule 5: Revert (Point 5)
- Release second paddle (first still held) → output reverts to first element type
- Implemented by checking held paddle state when no memory active

### Rule 6: Re-closure / Flick (Point 6)
- Release and re-press first paddle → reactivates first memory
- Causes new seizure back to first element type
- Each rising edge creates new memory flag

### Rule 7: Last-Touch Priority (Point 7)
- Most recently pressed paddle controls output
- Until opposite paddle pressed OR first paddle released

## 4. Algorithm Implementation

### State Variables

```c
// Memory flags - set on rising edge of paddle
bool dit_memory = false;
bool dah_memory = false;

// Sequencer state machine
typedef enum {
    IDLE,           // Waiting for next element decision
    SENDING_DIT,    // Sending dit (1 unit = 80ms @ 15 WPM)
    SENDING_DAH,    // Sending dah (3 units = 240ms @ 15 WPM)
    SPACING         // Inter-element space (1 unit = 80ms)
} KeyerState;

KeyerState state = IDLE;
uint8_t units_remaining = 0;

// Previous paddle states for edge detection
bool prev_dit_paddle = false;
bool prev_dah_paddle = false;
```

### Main Sequencer Function (called every 80ms)

```c
void ultimatic_sequencer_tick(void) {
    // Read current paddle states
    bool dit_paddle = read_dit_paddle();
    bool dah_paddle = read_dah_paddle();
    
    // POINT 1 & 7: Detect rising edges - "whenever a key makes contact"
    // Set memory flag on paddle press (rising edge detection)
    if (dit_paddle && !prev_dit_paddle) {
        dit_memory = true;  // Rising edge - paddle just pressed
    }
    if (dah_paddle && !prev_dah_paddle) {
        dah_memory = true;  // Rising edge - paddle just pressed
    }
    
    // State machine
    switch (state) {
        case IDLE:
            // Decision: which element to send next
            // Priority: memory flags (seizure) > held paddles (continuous)
            
            // POINT 3: Check memory flags first (seizure mechanism)
            if (dah_memory) {
                key_down();
                state = SENDING_DAH;
                units_remaining = 3;
                dah_memory = false;  // Clear after starting
            }
            else if (dit_memory) {
                key_down();
                state = SENDING_DIT;
                units_remaining = 1;
                dit_memory = false;  // Clear after starting
            }
            // POINT 2 & 5: No memory, check held paddles (continuous/revert)
            else if (dah_paddle) {
                key_down();
                state = SENDING_DAH;
                units_remaining = 3;
            }
            else if (dit_paddle) {
                key_down();
                state = SENDING_DIT;
                units_remaining = 1;
            }
            // else: stay IDLE (both paddles released, no memory)
            break;
            
        case SENDING_DIT:
        case SENDING_DAH:
            // Count down element duration
            units_remaining--;
            
            if (units_remaining == 0) {
                // Element complete - transition to spacing
                key_up();
                state = SPACING;
                units_remaining = 1;  // One unit of spacing
            }
            break;
            
        case SPACING:
            // Count down spacing duration
            units_remaining--;
            
            if (units_remaining == 0) {
                // Spacing complete - transition to idle
                state = IDLE;
                // Next tick will check memory flags and paddle states
            }
            break;
    }
    
    // Store current states for next tick's edge detection
    prev_dit_paddle = dit_paddle;
    prev_dah_paddle = dah_paddle;
}
```

### Key Implementation Notes

1. **Rising Edge Detection:** Critical for Points 1, 6, and 7. Every paddle press creates a memory flag, including re-closures.

2. **Priority Order in IDLE:**
   - First check memory flags (enables seizure)
   - Then check held paddles (enables continuous/revert)

3. **Memory Clearing:** Memory flag cleared when element transmission STARTS, not when element completes.

4. **Revert Mechanism:** Automatically handled by checking held paddles when no memory active (Point 5).

5. **Re-closure (Flick):** Each new rising edge sets memory flag again, enabling Point 6 behavior.
